# CardTracker

CardTracker is a local-first inventory dashboard for a single collector. The desktop app stores inventory in a local SQLite database on the user's machine and keeps the Card Ladder CSV import workflow for bulk updates.

## Stack

- React + TypeScript + Vite
- Electron for packaged desktop delivery
- Local SQLite storage powered by `sql.js`
- Card Ladder CSV preview + repeatable import

## Local setup

1. Install Node 20 or newer.
2. Run `npm install`.
3. Run `npm run build`.
4. Run `npm run electron:start`.

The app data is stored locally in Electron's `userData` directory. Typical locations:

- macOS: `~/Library/Application Support/CardTracker/cardtracker.db`
- Windows: `%AppData%/CardTracker/cardtracker.db`

## Development

The current app expects to run inside Electron because the renderer talks to the local database through a preload bridge.

- `npm run typecheck`
- `npm run build`
- `npm run electron:start`

## Data model

Inventory records keep the same fields as before:

```text
App ID,Date Purchased,Quantity,Player,Year,Set,Variation,Number,Category,Condition,Investment,Estimated Value,Ladder ID,Notes,Date Sold,Sold Price,Image,Created At,Updated At,Last Imported At
```

Category and condition suggestions are derived from the records already stored in the local database.

## CSV import

Card Ladder CSV imports are still supported. The import expects this exact header row:

```text
Date Purchased,Quantity,Player,Year,Set,Variation,Number,Category,Condition,Investment,Estimated Value,Ladder ID,Notes,Date Sold,Sold Price,Image
```

The app previews the import, flags invalid rows, and upserts matched records by `Ladder ID`.

## GitHub workflow

GitHub Actions:

- build the app on Windows
- on `v*` tags, package Electron desktop releases for Windows
- publish the generated installer and portable artifacts to GitHub Releases

## Friend-ready release

The final handoff path does not require your friend to install Node or npm.

Each GitHub Release contains Windows desktop artifacts generated by Electron Builder:

- Windows installer `.exe`
- Windows portable `.exe`

Your friend should:

1. Download the latest Windows GitHub Release artifact.
2. Install or open the packaged app.
3. Launch CardTracker like a normal desktop app.
4. Use the app locally. No Google account or external service setup is required.

## Packaging locally

Developer packaging flow:

1. Run `npm run build`
2. Run `npm run package:windows` on Windows

Artifacts are created in `release/`.
